# Page Archetypes

Reference courte pour savoir comment chaque type de page doit utiliser le systeme.

## Landing / social hub

- Densite: moyenne
- Largeur: large, hero respirant, cartes bento ou liens mis en avant
- Navigation: topbar simple, drawer compact
- Personnage: hero signature accepte
- Priorite: presence, hierarchie, liens principaux

Ne pas faire:

- un hero trop serre
- une navbar qui prend trois lignes sur mobile
- des cartes generiques de portfolio startup

## Tool dashboard

- Densite: forte mais lisible
- Largeur: exploiter le large ecran pour toolbar, KPIs, resultats et side panel
- Navigation: topbar compacte, drawer sur petit ecran
- Personnage: uniquement empty state, onboarding ou rappel secondaire
- Priorite: vitesse de lecture, action, resultat

Ne pas faire:

- gros visuel dans la zone dense
- accent pastel pour du texte critique
- panneaux trop etroits sur desktop large

## Library / collection

- Densite: moyenne a forte
- Largeur: side panel clair + grille media qui profite du large ecran
- Navigation: topbar compacte, drawer sur petit ecran
- Personnage: hero editorial ou rappel discret
- Priorite: filtres, decouverte, tags, cartes

Ne pas faire:

- filtres caches sans point d'entree clair
- cartes trop petites sur grand ecran
- surcharge decorative dans les zones de browse

## Auth

- Densite: faible a moyenne
- Largeur: split layout lisible, formulaire prioritaire, visuel secondaire
- Navigation: topbar minimale, drawer sur compact si necessaire
- Personnage: visuel lateral ou rappel doux
- Priorite: comprehension immediate, CTA net, friction basse

Ne pas faire:

- visuel plus important que le formulaire
- deux colonnes forcees sur petit ecran
- CTA secondaires plus forts que l'action principale

## Guide et docs

- Densite: structuree
- Largeur: tres large sur desktop, encore plus large si sidebar repliee
- Navigation: sidebar desktop + drawer compact
- Personnage: ponctuel uniquement
- Priorite: lecture, repere, navigation interne

Ne pas faire:

- doubler les navigations visibles en compact
- garder une largeur de lecture trop etroite sur grand ecran
- laisser la documentation sans source Markdown claire
